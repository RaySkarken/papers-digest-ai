# Архитектура

Проект построен как команда агентов с разными ролями и четким разделением ответственности.

## Роли агентов
- **Архитектор**: определяет источники, критерии качества и формат итогового поста.
- **Разработчик**: собирает статьи из источников, нормализует данные.
- **Аналитик**: ранжирует по релевантности запросу, делает краткие выводы и рекомендации.
- **Тестировщик**: проверяет качество результата (минимальные требования, крайние случаи).

## Поток данных
1. Архитектор формирует спецификацию под запрос.
2. Разработчик извлекает статьи за сегодня из источников.
3. Аналитик рассчитывает релевантность, формирует digest‑пост.
4. Тестировщик выполняет проверки и формирует отчет.

## Компоненты
- `papers_digest.sources`: адаптеры к источникам.
- `papers_digest.ranker`: ранжирование по релевантности.
- `papers_digest.llm`: опциональный LLM клиент.
- `papers_digest.agents`: реализация ролей и оркестрация.
- `papers_digest.cli`: интерфейс командной строки.
