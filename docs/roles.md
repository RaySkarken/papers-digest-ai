# Role split for the vibecoding team

This project uses a strong multi-agent team for development (not in-app agents).

## Analyst

- Clarifies the user query and defines ranking relevance criteria.
- Defines data sources, constraints, and output format.
- Produces acceptance criteria and success metrics.

## Architect

- Defines system modules and interfaces.
- Chooses tech stack and dependencies.
- Defines extensibility for new sources and summarizers.

## Developer

- Implements pipeline, source adapters, and CLI.
- Writes tests and keeps code quality.
- Maintains documentation and examples.

## Tester

- Creates test cases for date filtering, ranking, and formatting.
- Adds unit tests and CI workflow.
- Defines manual checklists and test data.
